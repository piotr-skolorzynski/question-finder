Czym jest let. Jaka jest różnica między let a var?

Wersja JavaScript ES6 wprowadziła słowa kluczowe let oraz const, które deklarują zmienną oraz stałą o zasięgu blokowym. Ich zasięg jest ograniczony do bloku czy też wyrażenia w którym są używane. Let dotyczy również hoisting ale nie są inicjowane wartością jak var z undefined. Wywołanie let przed jej deklaracją spowoduje błąd np. "error: info is not defined". Ponadto zdefiniowanie zmiennej let poza funkcją NIE dodaje jej do obiektu window.
Natomiast zmienne deklarowane poprzez var mają zasięg globalny chyba, że były zadeklarowane w funkcji przez co miały lokalny zasięg. Zmiennych var dotyczy hoisting, przez co są inicjalizowane wartością undefined automatycznie na początku funkcji. zdefiniowanie zmiennej var poza funkcją automatycznie dodaje ją do obiektu window. 

            let a = 10;
            if (1 == 1) {
                let a = 3;
                console.log(a); // 3
            }
            console.log(a); // 10

Czym są funkcje strzałkowe tzw. arrow (fat arrow, lambda)

Funkcja strzałkowa (ES6) to uproszczona funkcja nie posiadająca this, arguments czy super, nie może być konstruktorem i najlepiej sprawdza się jako skrócony zapis zwykłych funkcji np. przekazywanych jako argument.

            let add = (x,y) => x + y;
            let multiply = (a,b) => a * b;
            add(1,3); //4
            multiply(2,5); //10

            let arr = [9,8,7,6,5,4,3,2,1];
            arr.forEach(a => console.log(a)); // 9, 8, 7, 6 ...

Jaki jest najszybszy sposób na pobranie elementu z drzewa DOM?

Jeżeli mamy identyfikator elementu to użycie getElementById() będzie najszybsze, ale posiadanie zbyt wielu identyfikatorów nie jest najlepszym rozwiązaniem ze wględu na powtarzanie styli.
Klasy CSS są drugim najszybszym sposobem na wybranie elementu np. poprzez getElementByClassName().


    Poniżej ranking od najszybszej do najwolniejszej metody:

  • id np. #main
  • klasy np. .box
  • siblings: div + p
  • dziecko: div > p;
  • potomkowie (descendant): div span
  • uniwersalny selektor: *
  • atrybuty: div[data-type="gallery"]
  • pseudoklasy: div:first-child

Czy zdarzenia window.onload oraz document.onload są wywoływane jednocześnie?

window.onload jest wywoływany kiedy DOM jest gotowy razaem z załadowanymi obrazkami, css, skryptami itd, czyli kiedy wszystko jest załadowane.

document.onload jest wywoływany kiedy DOM jest gotowy i załadowany, ale np. obrazki nie zostały jeszcze do końca pobrane.


    Dla document istnieje zmienna document.readyState której wartość odzwierciedla postęp ładowania dokumentu, wartość:

  • "loading" gdy jest ładowany
  • "interactive" kiedy parsowanie się kończy, ale dalej ładowane są dalsze zasoby
  • "complete" kiedy został załadowany

Zdarzenie readystatechange jest wywoływane w dokumencie ze względu na powyższe zmiany.

Czym jest null oraz undefined?

Wartość null reprezentuje intencjonalny brak wartości w zmiennej, jest to jeden z prymitywów w JavaScript. Wartość null ma typ object zwracaną z funkcji typeof. Aby opróżnić zmienną z wartości stosuje się przypisanie null jako wartości.

undefinded jest przypisywane automatycznie do nowo utworzonej zmiennej jeśli nie została przypisana jakaś inna wartość. Typem undefined jest undefined.


Główne różnice pomiędzy null a undefined:

    null:

  • jest przypisywaną wartością określającą, że zmienna nie wskazuje na obiekt lub jakąś wartość. Oznacza pustkę, brak referencji
  • typ null to obiekt
  • jest prymitywem
  • konwertuje dię do 0 przy prymitywnych operacjach

    undefinded:

  • nie jest przypisywaną wartością, ale nowo powstałe zmienne bez przypisanej wartości będą miały undefined
  • typ undefined to undefined
  • undefined jest prymitywem określającym, że zmienna nie ma przypisanej wartości
  • reprezentuje brak wartości
  • konwertuje się do NaN przy prymitywnych operacjach

Jaka jest różnica między atrybutem a właściwością?

Atrybuty w tagu html są przeniesione do drzewa DOM jako właściwość attributes będąca kolekcją wszystkich atrybutów w danym elemencie html, jest to w praktyce NamedNodeMap i jest uzupełniana podczas parsowania kodu html.

Można również użyć funkcji na danym elemencie drzewa DOM getAttribute() do uzyskania wartości atrybutu oraz setAttribute do zmiany wartości.

Czym jest NaN?

NaN jest to globalna właściwość reprezentująca wartość "Not-a-Number". Wskazuje, że wartość nie jest dopuszczalną liczbą, rzadko się z niej korzysta, ale czasem się pojawia podczas niedozwolonych operacji matematycznych np. Math.sqrt(-1); zwróci NaN.

Czym jest isNaN?

Jest to funkcja sprawdzająca czy wartość jest niedozwoloną liczbą, zwraca true jeśli tak jest np. isNaN(Math.sqrt(-1); //true

Czym jest isFinite?

Jest to funkcja sprawdzająca czy wartość to skończona liczba np:

            isFinite(Infinity); // false
            isFinite(-Infinity); // false
            isFinite(NaN); // false
            isFinite(55); // true

Czym jest operator typeof?

Operator typeof w JavaScript zwraca typ danych przechowywanych w zmiennej np:

            console.log(typeof "text"); // string
            console.log(typeof 12); // number
            console.log(typeof true); // boolean
            console.log(typeof [1,2]); // object
            console.log(typeof {a:1}); // object
            console.log(typeof (1+2)); // number
            console.log(typeof undefined); // undefined
            console.log(typeof null); // object

Jak jest różnica między window a document?

Globalny obiekt window w przeglądarce ma w sobie wszystkie globalne zmienne, globalne funkcje, obiekt location, XMLHttpRequest, console, localStorage itd.

Obiekt document również jest zawarty w window jako jego właściwość i jest reprezentacją DOM (Document Object Model) powstały w wyniku parsowania kodu HTML. Tagi HTML przekształcane są na węzły - nodes, dlatego document dostarcza funkcje jak getElementById(), addEventListener(), getElementsByTagName(), itd.

            document === window.document; // true

window może być pominięty przy odwołaniu się do document ponieważ JS znajdzie sobie ten obiekt na końcu w scope chain.

Czym jest JSON?

JSON - JavaScript Object Notation jest tekstowym formatem danych na bazie składni JavaScriptu, jest stosowany w transmisji danych poprzez sieć. Mime type JSON-a to application/json, rozszerzenie pliku .json

Aby skonwertować JSON-a na obiekt JavaScript stosujemy:

            let obj = JSON.parse(text);

Aby skonwertować obiekt na JSON-a:

            let strJson = JSON.stringify(obj);
